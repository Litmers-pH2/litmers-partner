# 로그인 및 회원가입 관련 변경 요구사항 

## 1. 회원가입 화면

**참고 Figma**

- 회원가입 메인: https://www.figma.com/design/dGARLTFVGTnoQ8cijcLbke/Jyaguenfriends--Internal-?node-id=10-2875

### 1-1. 필수 입력 필드 구성

회원가입 화면에는 아래 필드들이 **위에서 아래 순서로** 배치된다.

1. 아이디(또는 이메일)
2. 비밀번호
3. 비밀번호 확인
4. 휴대폰 번호
5. 인증번호 입력
6. 회원가입 완료 버튼

---

### 1-2. 비밀번호 / 비밀번호 확인

- **비밀번호 확인 필드 필수**
- 비밀번호 입력 시 **실시간 규칙 체크 UI 표시**

비밀번호 규칙:

- 8자 이상
- 영문 포함
- 숫자 포함
- 특수문자 포함 (선택 조건)

동작 요구:

- 비밀번호 입력 중 실시간으로 각 조건 충족 여부 표시
- 비밀번호 확인 필드는 **비밀번호와 불일치 시 즉시 에러 상태**

---

### 1-3. 휴대폰 번호 인증

- 휴대폰 번호 입력 필드 + `인증번호 전송` 버튼 존재
- 인증번호 전송 후:
    - 인증번호 입력 필드 활성화
    - 인증번호 유효 시간 카운트 표시
- 인증번호가 일치해야 다음 단계 진행 가능

---

## 2. 회원가입 시 에러 케이스 처리

**참고 Figma**

- 에러 상태 예시 1: https://www.figma.com/design/dGARLTFVGTnoQ8cijcLbke/Jyaguenfriends--Internal-?node-id=8-2433
- 에러 상태 예시 2: https://www.figma.com/design/dGARLTFVGTnoQ8cijcLbke/Jyaguenfriends--Internal-?node-id=27-88

### 2-1. 공통 원칙

- 모든 필드는 **입력 중 / 포커스 아웃 / 제출 시** 에러 상태를 명확히 구분
- 에러 메시지는 **필드 하단에 인라인 텍스트로 노출**

### 2-2. 주요 에러 케이스

- 필수값 미입력
- 형식 오류 (이메일, 휴대폰 번호)
- 비밀번호 규칙 미충족
- 비밀번호 / 비밀번호 확인 불일치
- 인증번호 미입력 또는 불일치
- 이미 가입된 계정

## 6. 마이페이지 → 계정 설정 → 회원 탈퇴

**참고 Figma**

- https://www.figma.com/design/dGARLTFVGTnoQ8cijcLbke/Jyaguenfriends--Internal-?node-id=27-864

### 6-1. 페이지 목적

- 사용자가 **의도적으로, 충분히 인지한 상태에서** 회원 탈퇴를 진행하도록 유도

### 6-2. 페이지 구성

1. 상단 타이틀: `회원 탈퇴`
2. 탈퇴 유의사항 안내 박스
    - 개인정보 삭제
    - 게시글 처리 방식
    - 재가입 불가
    - 포인트/혜택 소멸
    - 되돌릴 수 없음 강조
3. 본인 확인 영역
    - 현재 비밀번호 입력
    - 확인 문구 입력 (`회원탈퇴`)
4. 동의 체크박스
    - “위 유의사항을 모두 확인했으며, 회원 탈퇴에 동의합니다.”
5. 회원 탈퇴 버튼
    - 기본 비활성
    - 모든 조건 충족 시 활성화
6. 하단 경고 문구
    - 탈퇴 후 복구 불가 안내

---

### 6-3. 탈퇴 버튼 활성 조건

- 비밀번호 입력됨
- 확인 문구 정확히 `회원탈퇴`
- 동의 체크박스 체크됨

---

## 공통 개발 유의사항

- 모든 상태(disabled / error / active)는 명확히 구분
- 문자열 비교 로직은 **정확한 매칭**
- 실제 API 호출 전 단계별 validation 필수
- 디자인은 기존 Design System 컴포넌트 재사용
